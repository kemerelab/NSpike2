Spike2 Matlab HOWTO

 

Overview

   &nbs= p;        Spike2 can export data to a running matlab process.  Currently the matlab process must be running on one of the data acquisition computers, but it wi= ll eventually be possible to run spike2_matlab on a non-data acquisition computer.  The RT-Linux machine cannot be used as the matlab computer, although= behavioral and position outputs from the RT-Linux computer can be sent to matlab.

 

Configuration File Setup

   &nbs= p;        One machine should be designated as the Matlab comp= uter.  To do so, add “MATLAB” = to the datatype line for that computer. E.g.:

 

datatype  &n= bsp;        snow        &= nbsp;       CONTINUOUS        &= nbsp; MATLAB

 

   &nbs= p;        You can also specify the initial types of data to be sent to matlab as follows:

matlab      &n= bsp;       CONTINUOUS      1

   &nbs= p;        specifies that continuous data from electrode 1 should be sent to matlab.

 

matlab      &n= bsp;       SPIKE        &= nbsp;        2

   &nbs= p;        specifies that spike data from electrode 2 should be sent to mat= lab

 

matlab              BEHAVIOR

   &nbs= p;        specifies that digital IO to/from spike_behav should be s= ent to matlab

 

matlab           &= nbsp; POSITION

   &nbs= p;        specifies that the tracked pixels from each position from should be sent to matlab

 

When you start up spike2 on that computer (snow in this example) the spike_matlab program will automati= cally be launched.

 

 

Selecting matlab output fr= om spike2

   &nbs= p;        The Master menu on the master computer has three matlab related menu items:

   &nbs= p;        Matlab Save On

   &nbs= p;        Matlab Save Off

   &nbs= p;        Matlab Settings

 

   &nbs= p;        Matlab Save On and Matlab= Save Off start or stop the sending of data to spike_matlab<= /span>.  Spike_matlab<= /span> saves the data in a circularly linked list of buffers whose size is given in incl= ude/spike_matlab.h (default 10 buffers of 10MB each).  Spike_matlab<= /span> will automatically cycle through the buffers, but note that if you save >= ; nbuffers * buffersize byt= es, you will lose data, as the first buffer will be overwritten.

   

   &nbs= p;        Matlab Settings brings up a dialog that lets you sele= ct which types of data and which electrodes are saved.  This is only available when Matlab saving is off.

 

Sending data to matlab

            To send data to matlab, start up a matlab process and run the matlab function matlabspikesocket located in the Modules subdirectory= .  This is a mex= function (source: Modules/matlabspikesocket.cpp= ) and you may need to recompile it for your system, which you can do by running t= he Modules/compilematlab.m script.  

 

The format for the function= is as follows:

[spike= data contdata posdata behavdata starttime] =3D = matlabspikesocket();

 

spiked= ata is a cell array where each element contains the spike times and waveforms f= or the electrode of that number.

contda= ta is a cell array where each element contains the continuous data for the electr= ode of that number.  The time of t= he first element is given by starttime

posdat= a is a structure array where each element contains the tracked pixels for a position frame

behavd= ata is an array with the timestamp and status of the specified output port

 

Note that each time you run= matlabspikesocket you should get one buffer from spike_matlab.  It is currently the users responsibility to run matlabspikesocket often enough to download all of the saved data.

 

I would suggest starting th= e save, stopping the save, and then running matlab spikesocket a few times to get the data.  Make sure to use new variables in = the call so that you don’t erase the data from the previous save.
